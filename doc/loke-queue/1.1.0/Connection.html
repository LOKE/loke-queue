Connection

new Connection(url, options)

Create a connection to a RabbitMQ server.

Once connected, it will emit a connect event.

If the connection process fails, an error event will be emitted.

queue(queueNameopt, optionsopt) → {Queue}

Receive messages from a queue.

You must call .stop() on the resultant queue.

Parameters:
Name Type Attributes Description
queueName String <optional>

The name of the queue to receive events through. If not provided, then a new queue is created. If you want to share a queue between processes, simply use the same string here.
options Object <optional>
Properties
Name Type Attributes Default Description
durable Boolean <optional>
 true

If true, the queue will survive RabbiqMQ broker restarts.
exclusive Boolean <optional>
 false

If true, scopes the queue to the connection.
autoDelete Boolean <optional>
 false

Automatically delete the queue when there are no consumers.
maxConcurrent Number <optional>
 100

Maximum number of messages for this consumer to receive from the queue which can be waiting acknowledgement. This is the maximum concurrency of handler(message) calls.
